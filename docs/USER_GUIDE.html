📖 功能概览

SPX Helper 是一款专为 Shopee 大数据开发工程师设计的 Chrome 扩展工具，旨在提高日常工作效率。

📅 今日日程

集成 Google Calendar，直接查看今日日程，支持 Mini 待办看板。

🔗 快速链接

一键访问 DataSuite 平台和开发工具，支持自定义分类和链接。

📝 记事本

iOS 风格的笔记管理，支持 Markdown 渲染，可关联待办事项。

🌐 HTTP 请求

内置 API 调试工具，支持多种认证方式，自动生成 ApiMart JWT。

💬 AI 助手

集成 Smart Agent，支持 SQL 修正、翻译、代码优化等多种功能。

🛠️ 开发工具箱

时间戳转换、JSON 格式化、正则测试、文本对比、Mermaid 图表等。

✅ 核心优势
  • 🚀 开箱即用，无需复杂配置
  • 💾 数据本地存储，安全可靠
  • 🎨 现代化 UI 设计，操作流畅
  • ⚡ 轻量级扩展，不影响浏览器性能
  • 🔧 高度可定制，支持个性化配置

💾 安装指南

方法一：拖拽安装（最简单）⭐

  1. 打开扩展管理页面
    在 Chrome 浏览器中访问 chrome://extensions/
  2. 启用开发者模式
    在页面右上角打开"开发者模式"开关。
  3. 拖拽安装
    直接将 SPX_Helper_v2.6.3.zip 文件拖拽到扩展管理页面。
  4. 完成安装
    扩展图标会出现在浏览器工具栏，点击即可使用！
✨ 推荐理由

拖拽安装最简单快捷，无需解压，一步到位！

方法二：解压安装

  1. 下载并解压
    下载 SPX_Helper_v2.6.3.zip 并解压到任意目录（建议放在不会删除的位置）。
  2. 打开扩展管理页面
    在 Chrome 浏览器中访问 chrome://extensions/
  3. 启用开发者模式
    在页面右上角打开"开发者模式"开关。
  4. 加载扩展
    点击"加载已解压的扩展程序"按钮，选择刚才解压的文件夹。
  5. 完成安装
    扩展图标会出现在浏览器工具栏，点击即可使用！
⚠️ 重要提示

如果使用方法二，请勿删除解压后的文件夹，否则扩展将无法正常工作。建议将文件夹放在 ~/Applications/Chrome Extensions/ 等固定位置。

安装后配置

  1. 固定扩展图标
    点击浏览器工具栏的拼图图标，找到 SPX Helper，点击图钉固定到工具栏。
  2. 配置 Google Calendar（可选）
    如需使用"今日日程"功能，请参考下方的 日历配置章节。

更新指南

更新扩展时，为确保数据安全，请按以下步骤操作：

重要提示

直接删除扩展再重装会导致所有数据丢失！请务必先备份数据。

推荐更新流程：

  1. 备份数据
    打开扩展 → 点击右上角设置图标 → 点击「导出数据」→ 保存 JSON 备份文件
  2. 移除旧版本
    访问 chrome://extensions/ → 找到 SPX Helper → 点击「移除」
  3. 安装新版本
    按照上方安装步骤，拖拽或加载新版本的 zip 文件
  4. 恢复数据
    打开扩展 → 点击右上角设置图标 → 点击「导入数据」→ 选择之前保存的备份文件
  5. 验证恢复
    检查记事本、待办事项、快速链接等数据是否恢复正常
小贴士

建议定期导出数据备份，以防意外丢失。备份文件可以在不同电脑间迁移数据。

备份包含的数据：

  • 所有记事本内容
  • 所有待办事项
  • 自定义快速链接和分类
  • HTTP 请求历史
  • 扩展设置（窗口模式等）
  • 日历配置

📅 Google Calendar 配置指南

⚠️ 重要说明

由于 Google Calendar API 需要 OAuth 2.0 认证，首次使用"今日日程"功能需要完成以下配置。这是一次性操作，配置完成后即可长期使用。

快速配置步骤

  1. 获取扩展 ID
    访问 chrome://extensions/，找到 SPX Helper，复制扩展 ID（形如：abcdefghijklmnopqrstuvwxyz123456）
  2. 访问 Google Cloud Console
    打开 Google Cloud Console - 凭据页面
  3. 创建或更新 OAuth 客户端
    • 如果已有 OAuth 客户端：编辑现有客户端，更新"应用 ID"为扩展 ID
    • 如果没有：创建新的 OAuth 客户端 ID，类型选择"Chrome 应用"，应用 ID 填入扩展 ID
  4. 配置 OAuth 同意屏幕
    添加测试用户（你的 Shopee 邮箱），作用域添加：https://www.googleapis.com/auth/calendar.readonly
  5. 启用 Calendar API
    在 API 库中搜索并启用 "Google Calendar API"
  6. 在扩展中点击授权
    打开 SPX Helper，切换到"今日日程"标签，点击"授权访问 Google Calendar"按钮
✅ 配置成功

授权成功后，扩展会自动加载今日日程。如果遇到问题，可以点击右上角的"⚙️ 配置"按钮查看详细配置页面。

详细配置文档

更详细的配置步骤和截图说明，请查看扩展包中的 OAUTH_SETUP_QUICK.md 文件。

❓ 常见问题

Q1: 扩展安装后图标不显示？

A: 点击浏览器工具栏的拼图图标（扩展管理），找到 SPX Helper，点击图钉图标将其固定到工具栏。

Q2: 日历授权失败怎么办？

A: 请检查以下几点：

  • 确认已在 Google Cloud Console 中正确配置 OAuth 客户端
  • 确认扩展 ID 与 OAuth 客户端的"应用 ID"一致
  • 确认已启用 Google Calendar API
  • 确认你的邮箱已添加为测试用户
  • 尝试清除浏览器缓存后重新授权

Q3: 代办事项数据会丢失吗？

A: 不会。所有数据都存储在浏览器的本地存储中（chrome.storage.local），只要不清除浏览器数据或卸载扩展，数据会一直保留。

Q4: 可以在多台电脑上同步数据吗？

A: 当前版本使用本地存储，不支持跨设备同步。如需在多台电脑使用，需要在每台电脑上分别安装和配置。

Q5: Code Helper 无法使用？

A: Code Helper 功能需要连接到 Smart Agent API。如果无法使用，可能是：

  • 网络连接问题（需要能访问 smart.shopee.io）
  • Agent 服务暂时不可用
  • 浏览器阻止了跨域请求（已在 manifest.json 中配置权限）

Q6: 如何更新扩展？

A: 下载新版本的 zip 包，解压后：

  1. 访问 chrome://extensions/
  2. 找到旧版 SPX Helper，点击"移除"
  3. 按照安装步骤加载新版本
  4. 所有本地数据（代办事项、自定义链接等）会自动保留

Q7: 扩展占用资源多吗？

A: SPX Helper 是轻量级扩展，只在你点击图标打开弹窗时才会运行，不会在后台持续占用资源，对浏览器性能影响极小。

Q8: 快速链接和代码片段的修改会保存吗？

A: 会的。所有的自定义配置（链接、分类、代码片段等）都会自动保存到本地存储，下次打开扩展时会自动加载。
