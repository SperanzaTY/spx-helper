## ✅ 核心功能测试（必须）

### 1. 文本选取开关 - 关闭功能 🔴
**问题**: 关闭后仍弹窗（已修复）

**测试步骤**:
```
1. chrome://extensions/ → 刷新扩展
2. 打开任意 FMS 页面
3. 打开扩展弹窗 → 实用工具 → App 工具 → API 数据溯源
4. 在"⚙️ 功能设置"区域，关闭"文本选取查找"开关
5. 回到页面，用鼠标选择任意文字或数字
```

**预期结果**:
- ✅ 不出现 "🔍 查找来源" 浮动按钮
- ✅ 不弹出任何面板或窗口
- ✅ 控制台日志: `⏸️ [SPX Helper] 文本选取功能已禁用`

---

### 2. 文本选取功能 - 开启状态 🟢

**测试步骤**:
```
1. 在设置中开启"文本选取查找"开关
2. 在页面上选择一个数字（如 279.816）
3. 点击浮动按钮 "🔍 查找来源"
```

**预期结果**:
- ✅ 出现浮动按钮
- ✅ 点击后显示数据来源面板
- ✅ 显示匹配的 API 请求

---

### 3. 响应字段高亮 🎨

**测试步骤**:
```
1. 选择文字并查找来源
2. 点击"查看响应"按钮
3. 查看新打开的窗口
```

**预期结果**:
- ✅ 顶部显示匹配字段路径（如 data.list[0].amount）
- ✅ JSON 中匹配的值被金黄色高亮
- ✅ 语法高亮正常（VS Code 风格）

---

### 4. API 过滤条件 📡

**测试步骤**:
```
1. 在设置中修改"API 过滤条件"为: /api/
2. 点击"💾 保存过滤条件"
3. 点击"🔄 刷新页面"
4. 查看控制台和"已捕获 API 请求"数量
```

**预期结果**:
- ✅ Toast 提示: 过滤条件已保存
- ✅ 刷新后拦截的接口数量增加
- ✅ 控制台日志: `⚙️ [SPX Helper] API 过滤条件已更新: ["/api/"]`
